1. Опишите своими словами разницу между функциями 
   setTimeout() и setInterval().
 
Для вызова функции через заданный интервал времени существуют два метода:
setTimeout - позволяет вызвать функцию один раз через определённый интервал времени;
setInterval - позволяет вызывать функцию регулярно, повторяя вызов через определённый 
интервал времени.
Рекурсивный setTimeout является более гибким методом и позволяет задавать задержку между 
выполнениями более точно, чем setInterval.
   

2. Что произойдет, если в функцию setTimeout() передать
   нулевую задержку? Сработает ли она мгновенно, и почему?

Передача нулевой задержки в функцию setTimeout() (или setTimeout(func)) не привет к ее 
мгновенному выполнению, но вызов func будет настолько быстрым, насколько это возможно. 
Вызов функции будет запланирован сразу после выполнения текущего кода.

   
3. Почему важно не забывать вызывать функцию clearInterval(),
   когда ранее созданный цикл запуска вам уже не нужен?

При передаче функции в setInterval или setTimeout в планировщике на неё создаётся внутренняя 
ссылка, которая сохраняется тех пор, пока планировщик к ней обращается. 
Для setInterval она сохраняется пока не будет вызван clearInterval. Пока эта ф-ция существует,
также существуют и ее внешние переменные, т.к. она ссылается на внешнее лексическое окружение.
Эти переменные могут занимать больше памяти, чем сама функция. Поэтому, регулярный вызов функции
стоит отменить, если он больше не нужен.
